# 知识库文件查重与整理 Skill

## 适用场景

用户有一个目录堆积了大量工作文件，需要查重、去版本、清理压缩包、重组目录结构。

## 启动规则（必须）

1. **如果用户未提供源路径和目标路径**，必须主动询问：
   > "请提供：
   > 1. **源文件夹路径**（需要整理的原始目录）
   > 2. **结果文件夹路径**（查重整理后的输出目录）
   > 
   > ⚠️ 本 Skill 全程对**结果文件夹**操作，源文件夹只读、不修改。"

2. **明确声明**：所有操作（复制、删除、移动、重组）均在结果文件夹中进行，不触碰源文件夹。

## 核心理念

**多轮递进，逐轮确认。** 每轮聚焦一种重复模式，每轮结束生成报告、等待用户确认后再继续。每轮操作前创建备份，支持回滚。

## 执行流程（五轮）

### 第一轮：MD5 完全重复

**识别**：字节级完全相同的文件（MD5 哈希一致）

**排除规则**（不做清理的）：
- 工具导出的资源文件：`ui-*`、`jquery*`、`*.woff2`、`*.eot`、`*.ttf` 等前端框架资源
- 专业软件数据文件：`*.sa3`、`*.sn3`、`*.dll` 等软件安装目录自带文件
- 规律：同一目录树的多个子目录中出现完全相同的非用户创建的资源文件 → 工具生成，保留

**处理**：每组相同 MD5 的文件只保留一份，其余移入 `_backup/round1/`

### 第二轮：文件名相似度版本链

**识别模式**（按优先级）：
1. 显式版本号：`V1.0` / `v2` / `_v3` / `V4.0`
2. 日期后缀：`9.23` / `3.25` / `20230724`
3. 版本修饰词：`原始版` / `本地版` / `最新版` / `修正版` / `完整版` / `最终版` / `定稿版`
4. 副本标记：`副本` / `(1)` / `备份` / `备份新`
5. 方括号修饰：`【最新版】` / `【改后】` / `【改后2】`

**处理**：按基名（去除以上标记后的文件名）分组，每组保留修改时间最新/文件最大的一个。

### 第三轮：激进版文件名归一化

**方法**：用归一化函数（去掉括号、版本号、日期、修饰词）将文件名映射到"基名"，同一基名的文件视为同一文档的不同版本。

```python
# 核心归一化逻辑
def normalize(name):
    base = remove_brackets(name)       # （）【】［］
    base = remove_version_tags(base)   # V1.0 v2.3 V4
    base = remove_dates(base)          # 20230724 9.23 3.25
    base = remove_qualifiers(base)    # 新 备份 副本 改后 最终版 ...
    return base.strip(' _-')
```

**处理**：按归一化基名分组，每组保留修改时间最新的一个。大小相同的组跳过（第一轮已处理）。

### 第四轮：压缩包清理

**识别**：
- 同目录下存在与压缩包同名的文件夹且包含文件 → 已解压，压缩包可删
- 软件安装包/工具类（`Axure*.rar`、`*ISO`、`*安装包*`）→ 不属于知识库，移出

**处理**：已解压的压缩包移入备份区；工具类软件包直接移出知识库目录。

### 第五轮：目录结构重组

**顶层分类原则**（抽象）：

| 分类 | 判据 | 示例 |
|------|------|------|
| **项目** | 服务于特定交付物的完整工作集：系统设计、技术规格、需求文档、测试方案、外部合作项目的全套产物 | 系统设计文档、客户项目全套交付件 |
| **管理** | 跨项目的重复性管理产物：工作计划与周报、绩效考核、资质申报、业务分析、竞对研究、宣传材料 | 周报、考核表、资质材料、竞品分析 |
| **其他** | 无法归入以上两类的零散文件 | |

**子分类原则**：
- 项目类按具体项目分子目录
- 管理类按职能分子目录（计划考核、业务运营、宣传资料、人力资源、杂记等）
- 工具软件不进入知识库

## 每轮输出规范

每轮结束后必须：

1. **在结果目录根下生成报告**：`清理报告_R{轮次}_{轮次名称}.md`
2. **列出明细**：处理了哪些文件/目录，保留了什么，移除了什么
3. **询问用户**："第 N 轮完成。请审核结果。确认后进入下一轮，或有疑问我调整。"
4. **用户确认后才进入下一轮**

## 备份与回滚

- 每轮操作前，将待修改的文件复制到 `_backup/round{N}/`
- 用户要求回滚时，从 `_backup/` 恢复
- 全部完成后，用户确认，删除 `_backup/` 及所有临时文件

## 最终输出

完成后，结果目录根下生成：
- `清理报告_R1_MD5完全重复.md` ~ `清理报告_R5_目录结构重组.md` — 各轮明细
- `综合清理报告.md` — 汇总：总览、各轮摘要、执行过程、最终目录结构、后续操作指引

## 经验规则

| 规则 | 说明 |
|------|------|
| 用脚本处理，避免 shell 中文路径编码问题 | 不同 OS 对中文路径处理不一致 |
| 先移后备，不要直接删除 | 移入 `_backup/` 而非 `rm` |
| 归一化要激进 | 保守的正则漏掉大量版本链 |
| 工具生成的不算重复 | 框架资源文件保留 |
| 同大小同名不同后缀 → 格式转换，保留一种 | `.doc` vs `.docx` 内容大概率相同 |
